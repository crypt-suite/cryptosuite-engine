CORE ARCHITECTURE & DOCUMENTATION

An intricate, under-the-hood analysis of the cryptographic mathematics, buffer manipulations, and heuristic algorithms powering this system.

Module 1: Mathematical & Transposition Ciphers

1. Dynamic Caesar Cipher

Core Concept: A monoalphabetic substitution cipher relying on modular arithmetic.

  • Dynamic Alphabet Generation: Unlike standard 26-letter implementations, this engine dynamically constructs the modulo base (up to 62 characters: A-Z, a-z, 0-9) based on UI checkbox states.
  • Mathematical Engine: For encryption, the engine computes E(x) = (x + s) mod L, where x is the character index, s is the shift, and L is the dynamic alphabet length.
  • Decryption Wrap-Around: To handle negative modulo values in JavaScript during decryption, the formula explicitly forces positive wrapping: D(x) = ((x - s) % L + L) % L.

2. Vigenère Cipher

Core Concept: A polyalphabetic cipher utilizing a repeating key to flatten frequency analysis.

  • Index Mapping: The engine iterates through the plaintext. If a character is alphabetical, it calculates its base offset (65 for Uppercase, 97 for Lowercase).
  • Key Pointer: A separate index j tracks the key character. It wraps using j % key.length. The key character's ASCII value is mapped to a 0-25 shift range.
  • Algorithm: The resulting character is computed as String.fromCharCode((c.charCodeAt(0) - base + shift) % 26 + base), strictly preserving case and ignoring symbols/spaces.

3. Affine Cipher (Coprime Verification)

Core Concept: A substitution cipher combining multiplication and addition. E(x) = (ax + b) mod 26.

  • Key 'A' Validation: Key A must be coprime to 26 (e.g., 1, 3, 5, 7, 9, 11...). The backend enforces this by attempting to calculate the Modular Multiplicative Inverse of A modulo 26. If the loop fails, decryption is mathematically impossible, and the API rejects the request.
  • Decryption Formula: D(x) = a⁻¹ * (x - b) mod 26. The JS engine implements a secondary wrap to prevent negative modulo resolution.

4. Rail Fence Matrix Algorithm

Core Concept: A transposition cipher that disrupts positional syntax.

  • Vector Tracking: The algorithm constructs a 2D array of dimensions [rails][text.length].
  • Reflection Bounds: As characters are populated into the matrix, the engine checks for the ceiling or floor rail. Upon hitting them, the vector flips, causing the exact zigzag path.
  • Decryption Reconstruction: To decrypt, the engine first maps the zigzag path with `*` placeholder tokens, fills them linearly, and traces the path one last time to read out the plaintext.

Module 2: Offensive Security & Cryptanalysis

1. Asynchronous Caesar Brute-Force

Core Concept: Exhaustive key-space searching paired with heuristic detection.

  • Non-Blocking Event Loop: The UI utilizes setInterval() to yield control back to the browser render thread between guesses, preventing UI lockups.
  • Heuristic Halting: The engine scans the lowercased output of each shift against an array of 80+ common English words. If a match is found, the loop automatically halts and flags the key.

2. The Mutation Dictionary Attack

Core Concept: A multi-layered password cracking simulator predicting human habits.

  • Layer 1 (File Input): Parses user-uploaded .txt dictionary files asynchronously.
  • Layer 2 (Heuristic Substitution Map): Feeds base words into a recursive generator mapping characters to Leetspeak.
  • Layer 3 (Pattern Injection): Appends predictable human suffixes (1, 123, 2023) and tests common keyboard geometric patterns (qazwsx).
  • Layer 4 (Raw Permutation Generator): Falls back to a yield* generator to systematically test every single character combination up to the max length.

Module 3: Secure Vault & Deep-Buffer Integrity

1. Split-Key Buffer Encryption

Core Concept: Two-factor physical file fragmentation via AES-256-CBC.

  • Key Derivation (PBKDF2): User passwords are never used directly as keys. The backend generates a secure salt and applies PBKDF2 with 100,000 iterations.
  • Cipher Initialization: A 16-byte random IV ensures identical files encrypt to entirely different ciphertexts.
  • Buffer Slicing: The final ciphertext buffer is severed in half and bundled with a UUID text file into a downloadable ZIP.

2. Granular SHA-256 Integrity Validation

Core Concept: Cryptographic hashing and tokenized array diffing.

  • File-Level Hashing: The entire buffer is passed through Node's SHA-256 to generate a signature of the absolute binary state.
  • Tokenized Word Hashing: The backend splits normalized text into a massive array and calculates the hash for every single word.
  • Array Diffing: During integrity checks, the backend maps the indices of mismatched hashes and returns them to the frontend to render targeted visual alerts.

Module 4: Visual Cryptography Sub-Pixel Engine

HTML5 Canvas Pixel Manipulation

Core Concept: 2-out-of-2 secret sharing scheme reliant on human optical overlay.

  • Pixel Expansion: Output canvases are scaled up (Width × 2, Height × 2) to fit noise data.
  • Boolean Assignment: Black pixels receive a random subpixel pattern for Share 1, and the bitwise complement for Share 2. White pixels receive identical patterns.
  • Logical OR Overlay: The code simulates transparencies stacking by applying a logical OR operation across the pixel data, causing the secret image to emerge.

Core Concept: Strict MVC-patterned simulation severing cryptographic logic from DOM updates.

  • State Isolation: Hardware variables are contained in an immutable enigmaState object.
  • Logic Separation: The core operates blindly to the DOM, returning an output payload.
  • Visualization Bridging: SVG matrix transformations calculate absolute coordinates to render electron flow.

  • Inverse Backward Pass: Simulates reverse flow by finding the index of the letter in the wiring array.

3. The Odometer Anomaly

Core Concept: Accurate emulation of the physical double-stepping flaw.

  • The Mechanical Flaw: Replicates the "Double Step" phenomenon caused by pawl-and-ratchet design, advancing the middle rotor twice in a row when triggered.
