📖 Core Architecture & Documentation

An intricate, under-the-hood analysis of the cryptographic mathematics, buffer manipulations, and heuristic algorithms powering this system.

Module 1: Mathematical & Transposition Ciphers

1. Dynamic Caesar Cipher

Core Concept: A monoalphabetic substitution cipher relying on modular arithmetic.

  • Dynamic Alphabet Generation: Unlike standard 26-letter implementations, this engine dynamically constructs the modulo base (up to 62 characters: A-Z, a-z, 0-9) based on UI checkbox states.
  • Mathematical Engine: For encryption, the engine computes E(x) = (x + s) mod L, where x is the character index, s is the shift, and L is the dynamic alphabet length.
  • Decryption Wrap-Around: To handle negative modulo values in JavaScript during decryption, the formula explicitly forces positive wrapping: D(x) = ((x - s) % L + L) % L.

2. Vigenère Cipher

Core Concept: A polyalphabetic cipher utilizing a repeating key to flatten frequency analysis.

  • Index Mapping: The engine iterates through the plaintext. If a character is alphabetical, it calculates its base offset (65 for Uppercase, 97 for Lowercase).
  • Key Pointer: A separate index j tracks the key character. It wraps using j % key.length. The key character's ASCII value is mapped to a 0-25 shift range.
  • Algorithm: The resulting character is computed as String.fromCharCode((c.charCodeAt(0) - base + shift) % 26 + base), strictly preserving case and ignoring symbols/spaces.

3. Affine Cipher (Coprime Verification)

Core Concept: A substitution cipher combining multiplication and addition. E(x) = (ax + b) mod 26.

  • Key 'A' Validation: Key A must be coprime to 26 (e.g., 1, 3, 5, 7, 9, 11...). The backend enforces this by attempting to calculate the Modular Multiplicative Inverse of A modulo 26. If the loop (a * x) % 26 === 1 fails to find an integer x, decryption is mathematically impossible, and the API rejects the request.
  • Decryption Formula: D(x) = a⁻¹ * (x - b) mod 26. The JS engine implements a secondary wrap to prevent negative modulo resolution: (aInv * (x - b + 26)) % 26.

4. Rail Fence Matrix Algorithm

Core Concept: A transposition cipher that disrupts positional syntax.

  • Vector Tracking: The algorithm constructs a 2D array of dimensions [rails][text.length]. It utilizes a dir (direction) variable initialized to 1 (down).
  • Reflection Bounds: As characters are populated into the matrix, the engine checks if (row === 0 || row === rails - 1). Upon hitting the ceiling or floor rail, the vector flips (dir *= -1), causing the exact zigzag path.
  • Decryption Reconstruction: To decrypt, the engine first maps the zigzag path with `*` placeholder tokens. It then fills the placeholders linearly with the ciphertext, and finally traces the zigzag path one last time to read out the plaintext.

Module 2: Offensive Security & Cryptanalysis

1. Asynchronous Caesar Brute-Force

Core Concept: Exhaustive key-space searching paired with heuristic detection.

  • Non-Blocking Event Loop: The UI utilizes setInterval() to yield control back to the browser render thread between guesses. This prevents UI lockups while updating the visual logs at dynamic Guesses/sec rates.
  • Heuristic Halting: Instead of dumping all 25 shifts and forcing the user to read them, the engine scans the lowercased output of each shift against an array of 80+ common English words. If .some() returns true, the loop automatically halts and flags the index as the cracked key.

2. The Mutation Dictionary Attack

Core Concept: A multi-layered password cracking simulator that predicts human password habits.

  • Layer 1 (File Input): Parses user-uploaded .txt dictionary files asynchronously via the FileReader API.
  • Layer 2 (Heuristic Substitution Map): Feeds base words into a recursive generator that maps characters to Leetspeak (a → @, s → $, e → 3). It generates all possible combinations (e.g., apple → @pple).
  • Layer 3 (Pattern & Suffix Injection): Takes the results and appends highly predictable human suffixes (1, 123, 2023, 99). It also tests common keyboard geometric patterns (e.g., qazwsx, zxcvb).
  • Layer 4 (Raw Permutation Generator): If heuristic layers fail, the code falls back to a yield* generator function to systematically test every single character combination (A-Z, 0-9, symbols) up to the specified max length.

Module 3: Secure Vault & Deep-Buffer Integrity

1. Split-Key Buffer Encryption (Vault)

Core Concept: Two-factor physical file fragmentation via AES-256-CBC.

  • Key Derivation (PBKDF2): User passwords are never used directly as AES keys. The backend generates a 16-byte cryptographically secure pseudo-random salt. It then applies PBKDF2 with 100,000 iterations to derive a perfect 32-byte key.
  • Cipher Initialization: A 16-byte random IV (Initialization Vector) ensures identical files encrypt to entirely different ciphertexts. Salt and IV are logged to secure_meta.json keyed by a UUID.
  • Buffer Slicing: The final ciphertext buffer is literally severed in half: encrypted.slice(0, mid) and encrypted.slice(mid). The archiver library bundles these fragments with the UUID text file into a downloadable ZIP.

2. Granular SHA-256 Integrity Validation

Core Concept: Cryptographic hashing and tokenized array diffing.

  • File-Level Hashing: The entire `fs.readFileSync` buffer is passed through Node's `crypto.createHash("sha256")` to generate a 64-character hex signature representing the file's absolute binary state.
  • Tokenized Word Hashing: If the buffer passes the `indexOf("\uFFFD")` text-encoding check, the backend normalizes the text (removing punctuation and line-breaks). It then splits the text into a massive array and calculates the individual SHA-256 hash for every single word.
  • Array Diffing: During integrity checks, if the global hash fails, the backend compares the historical `originalWordHashes[i]` against `currentWordHashes[i]`. It maps the indices of mismatched hashes and returns an array of `changedWords` to the frontend, which renders targeted red `` elements around the exact tampered tokens.

Module 4: Visual Cryptography Sub-Pixel Engine

HTML5 Canvas Pixel Manipulation

Core Concept: 2-out-of-2 secret sharing scheme reliant on human optical overlay.

  • Pixel Expansion: The original text is drawn to a hidden canvas and its ImageData is extracted. To fit the noise data, the output canvases are scaled up (Width × 2, Height × 2).
  • Boolean Subpixel Assignment: The engine iterates through the original pixel matrix. If a pixel is deemed "black" (RGBA threshold < 128), it is assigned a random subpixel pattern matrix (e.g., [1,0],[0,1]) for Share 1. Share 2 is assigned the bitwise complement of that matrix ([0,1],[1,0]). If the pixel is white, both shares receive the exact same pattern matrix.
  • Logical OR Overlay Simulation: When generating the overlay, the code simulates transparencies stacking on top of each other. It extracts the `ImageData` for both Share canvases and processes them via a simulated logical OR: const blackOverlay = black1 || black2;. If either subpixel is black, light is blocked, resulting in the secret image emerging from the noise.

Module 5: Enigma Machine Emulator (Electromechanical Cryptanalysis)

1. Component & State Architecture

Core Concept: A strict MVC-patterned simulation built purely in modern ES6 JavaScript, severing cryptographic logic from DOM updates.

  • State Isolation: The machine's hardware variables—rotor configurations (I-V), active positions, ring settings (Ringstellung), and plugboard hash maps (Steckerbrett)—are contained in an immutable enigmaState object.
  • Logic Separation: enigmacore.js operates entirely blindly to the DOM. It processes integer indices mathematically and returns a {outputLetter, signalPath} payload.
  • Visualization Bridging: enigmaviz.js ingests the path array, calculating absolute screen coordinates using SVG matrix transformations (getCTM()) to render physical electron flow.

2. The Cryptographic Pipeline & Mathematics

Core Concept: Index-based array mapping modeling electrical conductivity through physical rotating discs.

  • Character Indexing: The engine maps A-Z to integers 0-25 via base-offset logic: c.charCodeAt(0) - 65.
  • Forward Rotor Substitution: As the signal enters the right side of a rotor, the code accounts for the rotor's rotation offset AND the ring setting offset. The relative contact index is: shifted = (i + pos - ring + 26) % 26.
  • Inverse Backward Pass: After striking the fixed Reflector (UKW-B), the signal travels backward through the rotors. Because the wiring is fixed, the code simulates reverse electrical flow by calculating the index of the letter: rotor.wiring.indexOf(letter), and reverse-mapping the offsets.

3. The Odometer Anomaly (Double Stepping)

Core Concept: Accurate emulation of the physical pawl-and-ratchet mechanical flaw present in the historical Enigma.

  • Primary Stepping: The rightmost rotor steps universally upon every key press via state.positions[2] = (rp + 1) % 26.
  • Notch Triggering: Each historical rotor possesses a physical notch (e.g., Rotor I notches at 'Q'). The engine checks if the normalized index hits this notch: indexToChar((mp + 26 - state.rings[1]) % 26) === middle.notch.
  • The Mechanical Flaw: Due to the historical physical design of the stepping pawls, when the middle rotor reaches its notch, it engages the left rotor's ratchet. However, when it steps the left rotor, it simultaneously pushes itself forward. The engine accurately replicates this "Double Step" phenomenon, advancing the middle rotor twice in a row.

4. The SVG Sub-Pixel Routing Engine

Core Concept: Dynamic trigonometric DOM plotting to simulate real-time electron pathing.

  • Trigonometric Pin Detection: Rotors contain 26 contact pins. To find the exact screen coordinate of a signal hitting a pin, the engine utilizes sine and cosine functions: x = Math.sin(rad) * radius and y = -Math.cos(rad) * radius.
  • Matrix Transformations: Because rotors are rotated using CSS transforms, local coordinates do not match the global SVG canvas. The engine employs point.matrixTransform(gear.getCTM()) to convert localized physical pin locations to absolute drawing coordinates.
  • Asynchronous Signal Lacing: The glowing orange signal path is plotted using the SVG stroke-dasharray and stroke-dashoffset properties. An asynchronous sleep() loop pairs with CSS transitions to simulate continuous current flow through physical wires, rendering exact intersection points down to the specific letter typed.
